# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest release | ✅ |
| Previous minor | ✅ (critical fixes only) |
| Older versions | ❌ |

## Reporting a Vulnerability

**Do NOT open a public GitHub issue for security vulnerabilities.**

Please report them via one of these channels:

1. **[GitHub's private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability)** (preferred)
2. Email the maintainer directly (find contact in the GitHub profile)

### What to include

- Description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- Affected versions and platforms (Bukkit, Fabric, Forge, NeoForge)
- Any suggested mitigations

### Response timeline

- **48 hours** — Acknowledgement of receipt
- **7 days** — Initial assessment and severity rating
- **30 days** — Fix or mitigation plan

All confirmed vulnerabilities will be disclosed publicly via a GitHub Security Advisory
after a fix is released.
